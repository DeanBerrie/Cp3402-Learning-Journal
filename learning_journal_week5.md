# Week 6 Learning Journal

###  Learning Activities and Resources
- This week focused on creating a WordPress child theme and applying customisations to an existing site. I followed developer documentation to correctly set up the child theme, including creating the theme folder, configuring the style.css file, and using functions.php to enqueue styles from the parent theme.
- I implemented at least six visible customisations using CSS, experimenting with layout, colours, typography, and spacing to change the appearance of the site without modifying the parent theme directly.
- I also used the browser inspector tool to test changes in real time before applying them to my stylesheet, which helped speed up the development process and reduce errors.
- In addition, I created a LinkedIn post reflecting on a key learning from this subject so far, adapting my writing style for a more professional and public audience.

### Content Insights
- This week helped me understand the importance of child themes in WordPress development. I learned that child themes allow developers to safely customise a site without modifying the parent theme, which prevents changes from being lost during updates. This reinforced the idea of maintaining clean, modular, and maintainable code.
- Working with functions.php to enqueue styles showed me how WordPress handles dependencies between stylesheets. I learned that properly loading styles is essential to ensure that customisations override the parent theme correctly without breaking the site.
- Using the browser inspector was a valuable technique, as it allowed me to experiment with design changes in real time. This made it easier to test different ideas quickly and understand how CSS properties affect layout and styling before committing changes to code.
- I also gained a deeper understanding of how front-end customisation works within a CMS. Rather than editing core files, I was able to adapt the visual presentation through CSS, which highlighted the separation between structure, content, and design.
### Learning Insights
- This week reinforced that understanding why a process is used is just as important as knowing how to do it. Creating a child theme is not just a technical requirement, but a best practice that supports long-term maintainability and scalability of a project.
- I found that experimenting through trial and error, especially using the browser inspector, significantly improved my learning. Being able to instantly see the impact of changes helped me build a more intuitive understanding of CSS and how small adjustments can affect the overall design.
- Adapting my learning into a LinkedIn post made me think more critically about how I communicate technical knowledge. It required me to simplify concepts and present them clearly, which helped reinforce my own understanding while also considering how others might interpret my work.
- This week also showed me the value of separating development into layers (parent theme vs child theme). This approach made me more aware of writing code that is reusable, non-destructive, and easier to manage over time.
- Moving forward, I want to continue improving my ability to experiment efficiently and document what I learn, as this combination of practical work and reflection has been the most effective way for me to build deeper understanding.
