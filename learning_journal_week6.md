### Content Insights

A key focus this week was understanding more advanced development workflows, particularly how GitHub can be used in a full WordPress development pipeline rather than just basic version control.

Through my research, I explored how a typical workflow might begin in a local development environment using Docker. This allows developers to run WordPress, PHP, and a database in isolated containers, ensuring consistency across different team members’ setups. Using Docker helps avoid issues where code behaves differently depending on the local machine, which is especially important in team projects.

From there, I looked into how changes would move through different environments. After development locally, code can be pushed to GitHub, which then acts as the central repository. This is where automated workflows using GitHub Actions can be introduced. For example, when code is pushed, a workflow could automatically run checks or deploy the site to a staging environment.

The staging environment acts as a testing space that closely mirrors production, allowing the team to review changes before they go live. I also researched how cloud services such as AWS could be used to host these environments, providing scalable infrastructure for both staging and production sites.

Finally, once changes are tested and approved, they can be deployed to the production environment, which is the live site users interact with. This structured workflow (local → staging → production) helps reduce errors and ensures that updates are properly tested before release.

Overall, this gave me a better understanding of how modern web development workflows are structured, especially for CMS-based projects like WordPress, where managing environments and deployments is just as important as writing code.

### Learning Insights

This week highlighted the importance of how I approach learning rather than just what I learn. I found that starting with a broad, high-level understanding of workflows helped me make sense of more complex concepts without getting overwhelmed by technical detail.

I also realised that I learn more effectively when I connect new concepts to a bigger picture. Instead of treating tools like GitHub, Docker, or cloud services as separate topics, understanding how they fit together in a workflow made it easier to remember and understand their purpose.

Another key insight was that learning through a mix of group work and independent research is effective for me. The group setup tasks provided practical exposure, while researching workflows allowed me to deepen my understanding at my own pace.

Finally, I recognised the value of revisiting complex topics multiple times. Some of the concepts I explored this week are quite advanced, so returning to them later will be important for reinforcing my understanding and building confidence over time.
