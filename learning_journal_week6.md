### Content Insights
- This week gave me a better understanding of modern development workflows for WordPress projects. I explored how local development environments using Docker can ensure consistency across team members’ machines, allowing WordPress, PHP, and the database to run in isolated containers. This approach prevents issues like "it works on my machine" when moving code between systems.
- I learned how code moves from a local environment to a central GitHub repository, and how automated workflows via GitHub Actions can run checks or deploy to a staging environment. The staging environment acts as a testing space that mirrors production, enabling safe review of changes before they go live.
- I also researched cloud-based hosting (e.g., AWS) for staging and production environments, which provides scalable infrastructure. This structured local to staging to production—reduces errors and ensures updates are tested before release.
Comparing this to simpler setups, I realised the importance of integrating version control, containerization, and cloud deployments for professional web development.
