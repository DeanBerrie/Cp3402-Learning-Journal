### Content Insights
- This week gave me a much clearer understanding of how local development environments work and why they are important. Using Docker showed me how containerisation can simplify setup and ensure consistency across different systems, this is important to help remove the 'but it works on my machine' problem as you are working on a centralized system which is important for both solo and group work, as you don't want to complete a project on a windows machine, and when uploaded to your ubuntu server it doesn't work.
- I also gained more confidence working with databases in WordPress and understanding how dynamic content is generated from stored data. This involved learning how data is stored, retrieved, and displayed across pages. An advantage of this is flexibility, as content can be updated without changing code, while a disadvantage is the added complexity and need to manage performance carefully.
- Exploring Vagrant in comparison to Docker also showed me that there are multiple valid approaches to environment setup, each with their own trade-offs. Vagrant provides a full virtual machine, which offers strong consistency and closely mirrors a real server environment. However, it can be more resource-intensive and slower to run. In contrast, Docker uses containers, which are lighter and faster to start, making it more efficient for development. The disadvantage is that Docker can be more complex to configure and may not fully replicate all aspects of a full operating system.
